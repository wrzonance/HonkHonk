# Grid Render Bench Harness (ADR-009 Baseline) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Establish a Criterion bench that measures the real sound-grid `view()` construction + layout + rasterization at 50 / 200 / 500 tiles on BOTH the tiny-skia and wgpu renderers, recording committed baseline numbers — the regression gate ADR-009 requires before #13 / #92 canvas work resumes.

**Architecture:** A `criterion` bench under `benches/grid_render.rs` drives the production `sound_grid::view_grid()` through Iced 0.14's headless render machinery (`UserInterface::build` for layout, `UserInterface::draw` for primitive generation), then rasterizes: tiny-skia into a CPU `Pixmap` (always available), and wgpu into an offscreen texture when a headless GPU adapter exists (graceful skip otherwise). A small support module builds deterministic `SoundEntry` fixtures and renderer factories so the bench body stays tiny and the clippy `too-many-lines` (50) / `cognitive-complexity` (10) caps hold.

**Tech Stack:** Rust, Iced 0.14 (`iced_tiny_skia`, `iced_wgpu`, `iced_runtime::UserInterface`, `iced_renderer::fallback::Renderer`), `tiny_skia`, `wgpu` 27, Criterion 0.5.

## Global Constraints

- **File size: 400 lines max per file**, functions ≤ 50 lines (project override, stricter than global).
- **Clippy gate:** `cargo clippy --all-targets --all-features -- -D warnings` must pass clean. Benches are compiled by `--all-targets`, so the bench code is subject to: cognitive-complexity 10, too-many-arguments 5, too-many-lines 50, type-complexity 200 (`clippy.toml`).
- **Format gate:** `cargo fmt -- --check` must pass.
- **No `.unwrap()` / `panic!()` in non-test code** — bench code is test-adjacent (a `[[bench]]` harness, not shipped in the binary), so `.expect("…")` with a clear message is acceptable in the bench *only*; never in `src/`. Prefer `expect` over bare `unwrap` so failures are legible.
- **Error context chains** apply to `src/` code; this plan adds no `src/` logic.
- **New crate justification:** Criterion is a new dev-dependency. ADR-009 explicitly mandates "a bench harness"; Criterion is the de-facto Rust micro-benchmark standard (statistical sampling, outlier detection, baseline save/compare via `--save-baseline`). The Rust stdlib has no stable benchmark harness (`#[bench]` is nightly-only). This justification goes verbatim in the PR `## Design decisions`. Criterion is `Apache-2.0 OR MIT` — license-clean for `cargo deny`.
- **CI:** the bench must compile on `ubuntu-latest` (it is built by the lint job's `cargo clippy --all-targets`). It must NOT require a GPU at *compile* time, and at *run* time the wgpu path must degrade gracefully (skip with a printed note) when no adapter is present. CI does not currently run `cargo bench`; do not add a bench-run CI step (out of scope — the gate is run manually / locally per ADR-009).
- **Renderer type:** `iced::Renderer` resolves to `iced_renderer::fallback::Renderer<iced_wgpu::Renderer, iced_tiny_skia::Renderer>` because `iced`'s default features enable `wgpu` and the app additionally enables `tiny-skia` (confirmed via `cargo tree`). `view_grid()` returns `Element<'_, Message, iced::Theme, iced::Renderer>`. The bench must therefore produce an `iced::Renderer` (the fallback enum), selecting the backend via the `Primary` (wgpu) / `Secondary` (tiny-skia) variant.

## File Structure

- **Create `benches/grid_render.rs`** — Criterion `criterion_group!` / `criterion_main!` entry. Defines the benchmark functions (one group per renderer, parameterized over 50/200/500). Keep each Criterion closure ≤ a few lines by delegating to the support module. Target ≤ ~120 lines.
- **Create `benches/support/mod.rs`** — fixture builders (`make_sounds`, `make_grid_ctx`) + the headless render drivers (`render_tiny_skia`, `try_render_wgpu`) + renderer factories. Target ≤ ~280 lines. (Criterion benches can have a sibling module via `mod support;` declared in `grid_render.rs`; the file lives at `benches/support/mod.rs`.)
- **Modify `Cargo.toml`** — add `criterion` to `[dev-dependencies]` and a `[[bench]]` table with `harness = false`.
- **Modify `Cargo.lock`** — regenerated by cargo when the dep is added (commit it).
- **Create `docs/benchmarks/grid-render-baseline.md`** — the committed baseline numbers + how to reproduce + how to use it as the #13/#92 gate. This is the "baseline numbers recorded (committed)" acceptance item.

**Public API the bench consumes (already exists, do not change):**
- `honkhonk::ui::sound_grid::view_grid(sounds: &[&SoundEntry], playing: Option<&str>, grid: GridCtx<'a>) -> Element<'a, Message>`
- `honkhonk::ui::sound_grid::GridCtx<'a> { slots: &SlotMap, triggers: &[Option<String>; 20], shortcuts_active: bool, columns: usize, sound_meta: &SoundMetaStore }`
- `honkhonk::state::{SoundEntry, AudioFormat, SlotMap, SoundMetaStore}` — `SoundEntry` has public fields `{ id, name, path, format, duration_ms, category }`.
- `honkhonk::app::Message` (the message type the Element is generic over).

**Note on `id`:** `tile_view` parses `sound.id.get(..8)` as base-16 to pick a `Tone`. Fixtures must use 16-hex-char ids (e.g. `format!("{:016x}", n)`) so this path executes exactly as in production and never panics.

```rust
//! Bench support: deterministic fixtures and headless render drivers.

use std::path::PathBuf;

use honkhonk::state::{AudioFormat, SlotMap, SoundEntry, SoundMetaStore};
use honkhonk::ui::sound_grid::GridCtx;

const CATEGORIES: &[&str] = &["Honk", "Memes", "Reactions", "Voicelines", "Music", "SFX"];

/// Builds `n` deterministic sound entries. Ids are 16 hex chars so the
/// production `Tone::from_index(u64::from_str_radix(&id[..8], 16))` path in
/// `tile_view` runs exactly as it does at runtime.
pub fn make_sounds(n: usize) -> Vec<SoundEntry> {
    (0..n)
        .map(|i| {
            let category = CATEGORIES[i % CATEGORIES.len()];
            SoundEntry {
                id: format!("{:016x}", (i as u64).wrapping_mul(0x9e37_79b9_7f4a_7c15)),
                name: format!("{category} sound number {i}"),
                path: PathBuf::from(format!("/sounds/{category}/sound_{i}.ogg")),
                format: AudioFormat::Ogg,
                duration_ms: if i % 2 == 0 { Some(1_000 + (i as u64) * 37) } else { None },
                category: category.to_owned(),
            }
        })
        .collect()
}

/// Borrows entries into the `&[&SoundEntry]` slice `view_grid` expects.
pub fn sound_refs(sounds: &[SoundEntry]) -> Vec<&SoundEntry> {
    sounds.iter().collect()
}

/// Owns the per-grid context state (`SlotMap`, trigger labels, meta store) so a
/// `GridCtx` can borrow from it for the duration of a bench iteration.
pub struct GridFixture {
    slots: SlotMap,
    triggers: [Option<String>; 20],
    sound_meta: SoundMetaStore,
}

impl GridFixture {
    pub fn new() -> Self {
        Self {
            slots: SlotMap::default(),
            triggers: std::array::from_fn(|_| None),
            sound_meta: SoundMetaStore::default(),
        }
    }

    pub fn grid_ctx<'a>(&'a self, columns: usize) -> GridCtx<'a> {
        GridCtx {
            slots: &self.slots,
            triggers: &self.triggers,
            shortcuts_active: false,
            columns,
            sound_meta: &self.sound_meta,
        }
    }
}

impl Default for GridFixture {
    fn default() -> Self {
        Self::new()
    }
}
```

/// Builds the grid Element and runs Iced's layout + draw passes against the
/// provided renderer. This is the `view()`-construction + tessellation work
/// ADR-009 cares about. Rasterization is renderer-specific and done by callers.
fn layout_and_draw(
    sounds: &[&SoundEntry],
    grid: GridCtx,
    renderer: &mut iced::Renderer,

    let element: Element<'_, Message> = view_grid(sounds, None, grid);
    let bounds = Size::new(VIEW_W as f32, VIEW_H as f32);
    let mut ui = UserInterface::build(element, bounds, Cache::new(), renderer);

    let style = Style { text_color: Color::WHITE };

}

- [ ] **Step 2: Implement the tiny-skia driver**

```rust
/// Full tiny-skia render: layout + draw + CPU rasterization into a Pixmap.
/// Returns the top-left pixel so the optimizer keeps the raster.
pub fn render_tiny_skia(sounds: &[&SoundEntry], grid: GridCtx) -> u32 {
    let mut ts = iced_tiny_skia::Renderer::new(Font::DEFAULT, Pixels(DEFAULT_TEXT_SIZE));
    // The Element is generic over `iced::Renderer` (the fallback enum); the
    // Secondary arm IS the tiny-skia renderer, so draw lands in `ts`'s layers.
    {
        let mut renderer = iced::Renderer::Secondary(ts);
        layout_and_draw(sounds, grid, &mut renderer);
        ts = match renderer {
            iced::Renderer::Secondary(r) => r,
            iced::Renderer::Primary(_) => unreachable!("constructed Secondary"),
        };
    }

    let mut pixmap = tiny_skia::Pixmap::new(VIEW_W, VIEW_H).expect("alloc pixmap");
    let mut mask = tiny_skia::Mask::new(VIEW_W, VIEW_H).expect("alloc mask");
    let viewport = Viewport::with_physical_size(Size::new(VIEW_W, VIEW_H), SCALE);
    let damage = [iced::Rectangle::new(
        iced::Point::ORIGIN,
        Size::new(VIEW_W as f32, VIEW_H as f32),
    )];
    ts.draw(
        &mut pixmap.as_mut(),
        &mut mask,
        &viewport,
        &damage,
        Color::from_rgb(0.09, 0.08, 0.06), // theme bg, opaque
    );
    let px = pixmap.data();
    u32::from_le_bytes([px[0], px[1], px[2], px[3]])
}
```

> If moving the renderer in/out of the `Secondary` variant fights the borrow checker, an equivalent is to keep the renderer in a local `iced::Renderer` for the whole function and reach the inner `iced_tiny_skia::Renderer` via a `match &mut renderer { Secondary(r) => r, .. }` borrow for the `.draw(...)` raster call. Either is fine; the constraint is that the SAME renderer instance that received `draw` is the one rasterized.

- [ ] **Step 3: Implement the wgpu context + driver**

```rust
/// Headless wgpu context: device, queue, engine. `None` when no adapter exists
/// (e.g. CI without a GPU) so the bench skips the wgpu group cleanly.
pub struct WgpuCtx {
    device: wgpu::Device,
    queue: wgpu::Queue,
    engine: iced_wgpu::Engine,
    format: wgpu::TextureFormat,
}

pub fn init_wgpu() -> Option<WgpuCtx> {
    let instance = wgpu::Instance::default();
    let adapter = pollster::block_on(instance.request_adapter(&wgpu::RequestAdapterOptions {
        power_preference: wgpu::PowerPreference::HighPerformance,
        force_fallback_adapter: false,
        compatible_surface: None,
    }))
    .ok()?;
    let (device, queue) = pollster::block_on(adapter.request_device(&wgpu::DeviceDescriptor {
        label: Some("honkhonk-bench"),
        ..Default::default()
    }))
    .ok()?;
    let format = wgpu::TextureFormat::Rgba8UnormSrgb;
    let engine = iced_wgpu::Engine::new(&adapter, device.clone(), queue.clone(), format, None);
    Some(WgpuCtx { device, queue, engine, format })
}

/// Full wgpu render: layout + draw + present to an offscreen texture.
pub fn try_render_wgpu(sounds: &[&SoundEntry], grid: GridCtx, gpu: &WgpuCtx) -> u32 {
    let mut wr = iced_wgpu::Renderer::new(gpu.engine.clone(), Font::DEFAULT, Pixels(DEFAULT_TEXT_SIZE));
    let mut renderer = iced::Renderer::Primary(wr);
    layout_and_draw(sounds, grid, &mut renderer);
    wr = match renderer {
        iced::Renderer::Primary(r) => r,
        iced::Renderer::Secondary(_) => unreachable!("constructed Primary"),
    };

    let texture = gpu.device.create_texture(&wgpu::TextureDescriptor {
        label: Some("honkhonk-bench-target"),
        size: wgpu::Extent3d { width: VIEW_W, height: VIEW_H, depth_or_array_layers: 1 },
        mip_level_count: 1,
        sample_count: 1,
        dimension: wgpu::TextureDimension::D2,
        format: gpu.format,
        usage: wgpu::TextureUsages::RENDER_ATTACHMENT | wgpu::TextureUsages::COPY_SRC,
        view_formats: &[],
    });
    let view = texture.create_view(&wgpu::TextureViewDescriptor::default());
    let viewport = Viewport::with_physical_size(Size::new(VIEW_W, VIEW_H), SCALE);

    let mut encoder = gpu
        .device
        .create_command_encoder(&wgpu::CommandEncoderDescriptor { label: None });
    wr.present::<&str>(
        &mut gpu.engine.clone(),
        &gpu.device,
        &gpu.queue,
        &mut encoder,
        Some(Color::from_rgb(0.09, 0.08, 0.06)),
        gpu.format,
        &view,
        &viewport,
        &[],
    );
    gpu.queue.submit(Some(encoder.finish()));
    gpu.device.poll(wgpu::Maintain::Wait);
    1 // present is the measured work; a token result keeps it un-elided.
}
```

> The exact `present`/`Engine::new`/`request_device` signatures are pinned by `iced_wgpu-0.14.0` and `wgpu-27.0.1` in the registry — read them (`~/.cargo/registry/src/*/iced_wgpu-0.14.0/src/lib.rs` line ~165 for `present`, `engine.rs` line ~25 for `Engine::new`) and adapt argument names/types exactly. wgpu 27 may have changed `request_adapter`/`request_device` to return values vs. `Result`; match the locked version. Add `pollster` to `[dev-dependencies]` (`pollster = "0.4"`) for `block_on` — it is tiny, `MIT OR Apache-2.0`, and avoids pulling a full async runtime into the bench. If `gpu.engine` cannot be cloned, store the engine behind the renderer per call or reconstruct per iteration; the measured cost is dominated by present, so per-iteration engine setup is acceptable as long as it is OUTSIDE the timed closure (see Task 4 `iter_batched` with setup).

//! Render baseline bench for the sound grid (ADR-009).
//!
//! Measures the production `sound_grid::view_grid()` through Iced's headless
//! render path (layout + draw + rasterize) at 50 / 200 / 500 tiles on both the
//! tiny-skia and wgpu renderers. See `docs/benchmarks/grid-render-baseline.md`.

mod support;

use criterion::{criterion_group, criterion_main, BenchmarkId, Criterion};

use support::{
    init_wgpu, make_sounds, render_tiny_skia, sound_refs, try_render_wgpu, GridFixture,
};

const SIZES: [usize; 3] = [50, 200, 500];
const COLUMNS: usize = 5;

fn bench_tiny_skia(c: &mut Criterion) {
    let mut group = c.benchmark_group("grid_render_tiny_skia");
    for &n in &SIZES {
